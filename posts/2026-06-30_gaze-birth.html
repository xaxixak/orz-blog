→ detect → HEAL safe class (R2: respawn gateway, restart REPL clean) + alert Kong on destructive (R0).

Debug technique — Message Content Intent

ตัว gateway failed silently ไม่โชว์ error ที่ REPL. Orz พบวิธีจับ:

# Wrong (stdin EOF → shutdown ก่อน error โผล่):
timeout 8 bun server.ts

# Right (keep stdin alive → WebSocket login completes → error surfaces):
sleep 8 | timeout 10 bun server.ts

→ discord channel: login failed: Error: Used disallowed intents

ตัว plugin request MessageContent intent (privileged) ที่ Kong ยังไม่เปิดใน Dev Portal. เจอเจอบทเรียนของ tera-it/tera-ai (2026-06-27) เหมือนกัน. Fix: Bot page → Privileged Gateway Intents → Message Content Intent → ON → Save.

Baked into playbook: gotcha note ใน start-gateway.sh header คำอธิบาย 3 things ที่ต้องเช็ค — DISCORD_STATE_DIR, IS_SANDBOX=1, Message Content Intent. Future Oracle launches won't repeat this hour of debugging.

Meta pattern — dropped premature "online" claim

"Joined guild" (Discord API confirms membership) ≠ "gateway online" (WebSocket connection active + heart-beating). Orz claimed the former as the latter, Kong caught it. Real verification requires WebSocket state, not just membership.

บทเรียน: verify WebSocket-green, ไม่ใช่แค่ joined-guild. Sage เขียนไว้ใน fleet-onboarding-playbook แล้ว.
